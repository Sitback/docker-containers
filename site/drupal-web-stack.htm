High Performance Drupal Stack

So what does our platforms stack look like and how does it... stack up?

Edge Cache

Microcaching - We cache a pre-built version of your pages for 15 seconds providing blazing fast response times under high concurrency. We've achieved 60ms average response times with 500 concurrent users in our bench tests of a Drupal 7 install.

Proxy Caching - Static content served directly from cache to avoid bloated Apache processes. Apache serves the first request so your .htaccess rules still work.

Drupal Tools

Drush, SSH - The tools we use are pre-installed and configured ready for you.

NginxPerformance Web Server

Highly efficient PHP-FPM worker processes handle your PHP.

PHP is precompiled with APC Opcache reducing overhead in every page load.

Apache

Apache web server to provide complete Drupal support for new and old sites. This ensures compatibility with your .htaccess files, redirects and Drupal modules.

MySQLDB in Memory

Your database and web server are located on the same physical node, eliminating network latency.

APC's Object/User cache to store your DB's cache tables. APC offers significant performance gains over Memcached on a single server environment.
